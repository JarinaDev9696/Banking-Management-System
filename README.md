🏦 Banking Management System – SQL Mini Project
📌 Project Overview
This project is a Banking Management System developed using SQL Server Management Studio (SSMS). It is designed to manage and analyze banking data by creating a structured database with multiple related tables and executing real-world SQL queries.
The project consists of 4 main tables:
Customers
Accounts
Transactions
Loans
A total of 17 SQL queries have been implemented to perform different banking operations such as customer analysis, account balance tracking, transaction analysis, and loan management.
🔧 SQL Concepts Used
INNER JOIN & LEFT JOIN – Used to connect related tables and retrieve meaningful information.
Aggregate Functions – Used SUM(), AVG(), COUNT(), MIN(), and MAX() for data calculations.
GROUP BY & HAVING – Used for summarizing and filtering grouped data.
Subqueries – Used for complex data comparisons and analysis.
Window Functions – Implemented advanced functions like:
RANK()
DENSE_RANK()
LAG()
LEAD()
SUM() OVER()
ORDER BY & TOP N – Used for sorting and finding top records.
📊 Key Features
✔ Display customer account details with account balances
✔ Find top customers based on highest account balances
✔ Analyze deposits and withdrawals
✔ Track customer transaction history
✔ Identify customers with and without loans
✔ Calculate transaction statistics
✔ Perform customer ranking and running total analysis
🛠 Tools & Technologies Used
SQL Server Management Studio (SSMS)
SQL (Structured Query Language)
